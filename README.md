# Password Manager

A personal password manager hosted on AWS. All encryption and decryption happens
client-side in the browser — plaintext passwords never leave your device.

## Architecture

| Layer | Service |
|-------|---------|
| Frontend | S3 + CloudFront + Route53 (`sutton5050.com` dashboard, `/passwords` vault) |
| API | HTTP API Gateway |
| Backend | Two Lambda functions (Python 3.12) |
| Database | DynamoDB (on-demand billing) |
| Secrets | SSM Parameter Store (SecureString) |
| CI/CD | GitHub Actions with OIDC |

**Security model:**
- Master password is never stored or transmitted as plaintext
- Browser derives two keys from the master password using PBKDF2 (600k iterations):
  - **Verification hash** — sent to the API once at login to prove identity
  - **AES-256-GCM encryption key** — stays in browser memory only, never leaves
- DynamoDB stores only ciphertext blobs — useless without the master password
- JWT tokens (8-hour expiry) authorise all API calls after login

**Vault features:**
- Client-side search — filters entries instantly by site, username, or description
- Reveal/hide toggle and one-click copy for each password
- The root domain (`sutton5050.com`) is intentionally left free for a future dashboard

## Prerequisites

- AWS CLI configured (`aws configure`)
- Node.js 18+ and `npm install -g aws-cdk`
- Python 3.12+
- CDK bootstrapped in `eu-west-2` (run once per account/region):
  ```
  cdk bootstrap aws://902672427642/eu-west-2
  ```

## First-time deployment

### 1. Install CDK Python dependencies

```bash
cd infrastructure
pip install -r requirements.txt
```

### 2. Deploy the backend stack

```bash
cdk deploy PasswordManagerBackend
```

Note the `ApiUrl` output — you'll need it in step 4.

### 3. Write frontend config

Create `frontend/config.json` with the API URL from step 2:

```json
{ "apiUrl": "https://YOUR-API-ID.execute-api.eu-west-2.amazonaws.com" }
```

> **Note:** In CI this file is generated automatically. For local testing, create it manually.
> Add `frontend/config.json` to your `.gitignore` — it is environment-specific.

### 4. Deploy the frontend stack

```bash
cdk deploy PasswordManagerFrontend
```

### 5. Set your master password

Run the setup script once to store your password hash and generate secrets:

```bash
cd ..  # back to project root
python scripts/setup.py
```

This will:
- Prompt for your master password (min 8 characters)
- Store the PBKDF2 verification hash + encryption salt in DynamoDB
- Generate a random JWT secret and store it in SSM Parameter Store

### 6. Done

Visit [https://sutton5050.com/passwords](https://sutton5050.com/passwords) and log in with your master password.

## CI/CD (GitHub Actions)

Pushes to `main` automatically deploy both stacks in order:
1. Backend deploys
2. `frontend/config.json` is written with the live API URL
3. Frontend deploys

The workflow uses OIDC — no AWS keys stored in GitHub. The `AWS_ROLE_ARN` secret
must be set in GitHub → Settings → Secrets.

## Changing your master password

Re-run the setup script:

```bash
python scripts/setup.py
```

Existing entries remain accessible — the encryption salt is unchanged, only the
verification hash and JWT secret are rotated.

## Tearing down

```bash
# Empty the S3 bucket first
aws s3 rm s3://YOUR-BUCKET-NAME --recursive --region eu-west-2

# Destroy stacks
cd infrastructure
cdk destroy PasswordManagerBackend PasswordManagerFrontend --force

# Note: DynamoDB table has RemovalPolicy.RETAIN — delete manually if wanted:
# aws dynamodb delete-table --table-name YOUR-TABLE-NAME --region eu-west-2
```
